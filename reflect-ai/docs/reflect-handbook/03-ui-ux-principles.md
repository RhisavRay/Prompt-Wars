# 03 - UI & UX Principles

**Version:** 1.0.0  
**Status:** Active  
**Last Updated:** 28 June 2026  
**Audience:** Developers, Designers, AI Coding Agents

---

# Purpose

This document defines the user experience and visual design principles that guide Reflect.

Every interface, interaction, animation, and workflow should support the emotional experience Reflect is designed to create.

Users should leave Reflect feeling calmer than when they opened it.

---

# Design Philosophy

Reflect is not a productivity tool.

Reflect is not a social platform.

Reflect is not an AI showcase.

Reflect is a quiet personal space.

Every design decision should reinforce that feeling.

---

# Core Design Principles

## 1. Calm over excitement

Reflect should never compete for the user's attention.

Avoid:

- Flashy animations
- Bright distracting colours
- Excessive notifications
- Loud visual elements

Instead, create an environment that encourages slowing down.

---

## 2. Reduce cognitive load

Users should spend their mental energy reflecting—not learning the interface.

Every screen should feel obvious.

Every interaction should feel natural.

If a feature requires explanation, simplify it.

---

## 3. Writing comes first

Everything exists to support writing.

The journal editor is the heart of Reflect.

The interface should never distract from the act of reflection.

---

## 4. Progress over perfection

Reflect is not about writing perfect journals.

The application should celebrate consistency, honesty, and growth.

Never pressure users into writing more than they want.

---

## 5. Respect emotional vulnerability

Users may write during difficult moments.

Every interaction should feel supportive rather than demanding.

Avoid guilt-driven language.

Avoid urgency.

Avoid pressure.

---

# Emotional Check-In

Reflect does not ask:

> "What is your mood?"

Reflect asks:

> "How are you feeling as you begin today?"

This distinction matters.

The check-in is not intended to perfectly classify emotions.

It simply captures the user's own perception of where they are beginning.

---

## Emotion Selection

Emotion selection should feel lightweight.

Users should never feel pressured to choose the "correct" emotion.

The interface should communicate:

> Choose the feeling that feels closest to where you're starting today.

There are no wrong answers.

---

## Emotion Chips

Emotion selection should use chips instead of dropdowns whenever practical.

Each chip should contain:

- Emoji
- Emotion name

Example:

😊 Joy

😌 Peace

😟 Anxiety

❤️ Gratitude

The emoji makes the interface warmer.

The text removes ambiguity.

Neither should exist without the other.

---

# AI Visibility

The AI should feel invisible.

Users should never feel like they are interacting with an AI chatbot.

Instead:

- Users write.
- AI quietly observes.
- Insights appear naturally when appropriate.

The AI should support reflection without demanding attention.

---

# Motion Design

Animations should guide attention.

They should never exist purely for decoration.

Motion should communicate:

- Focus
- Continuity
- Feedback
- Hierarchy

Avoid unnecessary movement.

Respect users who prefer reduced motion.

---

# Visual Language

Reflect should feel:

- Calm
- Warm
- Spacious
- Human
- Minimal

Avoid interfaces that feel:

- Corporate
- Clinical
- Busy
- Overwhelming

Whitespace is intentional.

Silence is part of the design.

---

# Typography

Typography should prioritize readability.

Long-form writing should remain comfortable for extended sessions.

Large blocks of text should feel inviting rather than intimidating.

---

# Colour Philosophy

Colours should support emotion rather than dominate it.

Neutral colours should remain the foundation.

Accent colours should guide attention—not overwhelm it.

Colour should never become the primary communicator of meaning.

---

# Accessibility

Reflect should be usable by everyone.

Every feature should support:

- Keyboard navigation
- Screen readers
- Visible focus states
- Sufficient colour contrast
- Responsive layouts

Accessibility is not optional.

---

# Cursor Behaviour

Every interactive element must communicate that it is interactive.

Buttons

Links

Cards

Menus

Avatars

Navigation

should all provide appropriate hover and cursor feedback.

Disabled elements should clearly communicate that they are unavailable.

---

# Loading Experience

Waiting should feel intentional.

Whenever possible:

- Show skeleton loaders
- Preserve layout stability
- Avoid flashing content
- Avoid blocking the interface unnecessarily

Loading should reduce uncertainty.

---

# Empty States

An empty state is an opportunity—not an error.

The application should encourage the user to begin rather than remind them that nothing exists.

Good empty states should feel welcoming.

---

# Error Messages

Errors should be calm.

Never blame the user.

Never expose technical details.

Good example:

"We couldn't save your journal right now. Please try again."

Poor example:

"Firestore write failed."

---

# Success Messages

Celebrate quietly.

Avoid excessive celebration.

Reflect should acknowledge progress without becoming distracting.

---

# AI Insights

AI insights should always be presented as observations—not facts.

Use language such as:

"You may have..."

"It seems..."

"Your writing suggests..."

Never:

"You are..."

"You definitely feel..."

"The AI determined..."

Reflect offers perspective.

It does not define the user.

---

# The Reflection Experience

The ideal journaling flow is:

Initial Check-In

↓

Write naturally

↓

Save immediately

↓

Continue living

↓

AI reflects quietly

↓

Insights become available later

Reflection should never feel interrupted by technology.

---

# Design Test

Before implementing any UI, ask:

- Does this reduce cognitive load?
- Does this help the user reflect?
- Does this feel calm?
- Would this still feel appropriate during someone's worst day?
- Is the interface supporting the user—or competing with them?

If any answer is "No", redesign before implementing.

---

# Guiding Statement

> Reflect should feel less like using an application...

> ...and more like entering a quiet room where thinking becomes easier.
