# Purpose

This document defines the shared vocabulary used throughout Reflect.

Every technical discussion, product discussion, handbook document, and implementation should use these terms consistently.

Whenever a new concept is introduced into Reflect, it should be added to this glossary.

---

# Product Terms

## Reflect

The application itself.

Reflect is an AI-assisted journaling platform focused on helping users understand how their thoughts and emotions evolve over time.

---

## Reflection

The overall process of:

- Checking in
- Writing
- Thinking
- Discovering insights

Reflection is broader than journaling.

---

## Journal Entry

A single written record created by the user.

A journal entry contains:

- Title
- Content
- Initial Check-In
- Creation timestamp
- Update timestamp
- Future AI Reflection

---

## Journal Session

One complete interaction with Reflect.

Typically consists of:

Initial Check-In

↓

Writing

↓

Saving

↓

Background AI analysis

↓

Future insights

---

# Emotional Terms

## Initial Check-In

Chosen entirely by the user before writing.

Represents how the user believes they feel as they begin writing.

It is never modified by AI.

---

## Emotion Chip

The UI element used during the Initial Check-In.

Each chip contains:

- Emoji
- Emotion name

Example:

😊 Joy

😟 Anxiety

❤️ Gratitude

---

## Emotional Journey

The movement of emotional tone throughout the writing process.

Reflect values emotional movement more than emotional labels.

---

## Emotional Shift

A meaningful change in emotional tone identified during AI Reflection.

Example:

Anxious

↓

Hopeful

---

# AI Terms

## AI Reflection

The collection of observations generated after a journal has been saved.

AI Reflection may include:

- Summary
- Themes
- Primary emotion
- Secondary emotion
- Emotional shift
- Reflection prompts

---

## Primary Emotion

The emotion most strongly reflected within the journal according to AI analysis.

---

## Secondary Emotion

A meaningful supporting emotion identified by AI.

---

## Tertiary Emotion

A less dominant emotional theme identified by AI.

---

## Theme

A recurring topic discussed within one journal.

Examples:

- Career
- Family
- Health
- Relationships
- Education

Themes are different from emotions.

---

## Insight

A meaningful observation discovered across multiple journal entries.

Insights develop over time.

Single journals generate reflections.

Multiple journals generate insights.

---

## Reflection Prompt

A thoughtful question generated by AI to encourage deeper self-reflection.

Reflection prompts should invite exploration rather than provide answers.

---

## Journal Memory

A structured, immutable AI representation of a single journal entry.

Journal Memory captures Reflect's understanding of one journal at the time it was written and never changes after creation.

---

## Active User Memory

Reflect's continuously evolving understanding of the user's current context.

Unlike Journal Memory, Active User Memory changes over time as journals are created, updated, or deleted.

---

## Archive Memory

Older Journal Memories that are no longer part of Active User Memory but remain available for retrieval when relevant.

Archive Memory preserves historical continuity while keeping AI context efficient.

---

# Technical Terms

## Dashboard

The authenticated home screen displaying journals and future insights.

---

## Service Layer

The portion of the application responsible for communicating with external systems such as Firebase and AI providers.

Business logic should not bypass this layer.

---

## Custom Hook

A reusable React hook encapsulating application logic.

Hooks should contain behaviour rather than presentation.

---

## Protected Route

A route requiring authentication before access is granted.

---

## Firestore

Reflect's primary database.

Firestore stores:

- User profiles
- Journal entries
- Future AI reflections

---

# Design Terms

## Calm Interface

An interface designed to reduce cognitive load rather than demand attention.

Reflect should always strive for calmness.

---

## Empty State

The experience presented when no content exists.

An empty state should encourage users to begin rather than remind them that nothing exists.

---

## Skeleton Loader

A placeholder shown while content is loading.

Skeletons reduce perceived waiting time while maintaining layout stability.

---

# Quality Terms

## Definition of Done

The minimum quality required before a feature is considered complete.

Includes:

- Working implementation
- Lint passing
- Build passing
- Manual testing
- UX verification

---

## Regression

An unintended failure of previously working functionality.

Every implementation should minimize regressions.

# Naming Guidelines

When introducing new terminology:

- Prefer simple language.
- Avoid unnecessary technical jargon.
- Use one term consistently throughout the project.
- Avoid synonyms that describe the same concept.

Consistency improves both user understanding and code quality.

---

# Guiding Statement

> Shared language creates shared understanding.

Every contributor should use the same words to describe the same ideas.
