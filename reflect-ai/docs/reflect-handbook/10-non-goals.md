# 10 - Non-Goals

**Version:** 1.0.0  
**Status:** Living Document  
**Last Updated:** 28 June 2026  
**Audience:** Everyone

---

# Purpose

This document defines what Reflect intentionally is **not**.

These non-goals protect the product from feature creep and ensure that every future decision remains aligned with Reflect's philosophy.

Whenever a proposed feature conflicts with this document, the feature should be reconsidered before implementation.

Reflect grows by becoming better—not by becoming everything.

---

# Product Philosophy

Reflect exists to help people understand themselves.

It does not exist to maximize engagement, increase screen time, or become another social platform.

Every feature should strengthen reflection.

Nothing should compete with it.

---

# Reflect Is Not A Social Network

Reflect is a private space.

It is not designed for:

- Followers
- Likes
- Comments
- Public profiles
- Viral content
- Social feeds
- Social validation

Reflection should never become performance.

---

# Reflect Is Not A Chatbot

Although Reflect uses AI, the AI is not the product.

Users should never feel that they are chatting with an assistant.

The primary interaction is writing.

AI quietly supports that process.

---

# Reflect Is Not A Therapist

Reflect should never:

- Diagnose mental health conditions.
- Replace professional care.
- Recommend treatments.
- Claim psychological expertise.

Reflect encourages self-reflection.

It does not provide healthcare.

---

# Reflect Is Not A Productivity Tool

Reflect does not exist to maximize:

- Streaks
- Daily active usage
- Time spent
- Notifications
- Habit pressure

Consistency is encouraged.

Pressure is not.

---

# Reflect Is Not Trying To Keep Users Inside The App

The ideal user journey is:

Open Reflect.

↓

Write honestly.

↓

Save.

↓

Leave.

If users spend less time in the application because reflection became easier, Reflect has succeeded.

---

# Reflect Does Not Judge

Reflect never labels users.

Reflect never scores emotions.

Reflect never ranks journal quality.

Reflect never defines success or failure.

Users should always feel accepted.

---

# AI Never Has The Final Word

AI observations are suggestions.

The user's own interpretation always takes priority.

Reflect never treats AI-generated content as objective truth.

---

# Reflect Does Not Manipulate Behaviour

Reflect should never use:

- Dark patterns
- Manipulative notifications
- Artificial urgency
- Fear of missing out
- Engagement tricks
- Addictive mechanics

Trust is more valuable than engagement.

---

# Privacy Is Not A Feature

Privacy is a foundation.

User journals should never become:

- Advertising data
- Recommendation data
- Public content
- Training data without explicit consent

Users own their reflections.

Reflect simply protects them.

---

# Simplicity Is A Requirement

Features should not be added simply because they are technically possible.

Before adding anything new, ask:

- Does this improve reflection?
- Does this reduce cognitive load?
- Does this align with Reflect's philosophy?

If not, it probably does not belong.

---

# AI Should Never Replace Reflection

The purpose of AI is to encourage deeper thinking.

Not to think for the user.

Reflect should ask thoughtful questions more often than it gives confident answers.

The best AI response is often curiosity—not certainty.

---

# Reflect Is Built For Years, Not Minutes

Reflect is not designed to impress users during the first five minutes.

It is designed to become more valuable after five months and five years.

Long-term trust matters more than short-term excitement.

---

# Every Feature Must Earn Its Place

A feature should only be added if it clearly strengthens the mission of Reflect.

Features are not added because competitors have them.

Features are added because they help users better understand themselves.

---

# The Final Test

Before implementing any feature, ask:

1. Does this help people reflect?
2. Does this respect the user's autonomy?
3. Does this preserve privacy?
4. Does this reduce rather than increase cognitive load?
5. Would we still build this if no competitor had it?
6. Will this still feel valuable five years from now?

If several answers are "No", the feature likely does not belong in Reflect.

---

# Guiding Statement

> Reflect is not trying to become the smartest journaling application.

> Reflect is trying to become the most trusted one.

Technology will evolve.

Models will improve.

Interfaces will change.

Trust is the one thing that must never be compromised.
